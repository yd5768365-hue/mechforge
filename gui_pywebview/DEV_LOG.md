# MechForge AI GUI 开发日志

## 2026-03-10

### 优化：知识库背景动态元素 - 工业真实感

**发现者**: Qwen Code

**问题描述**:
- 网格线条过于均匀对称，缺少工业真实感
- 粒子效果像"梦幻星尘"，与锻造主题不符
- 缺少动态元素暗示档案库正在被维护

**解决方法**:
1. **网格升级** (`css/industrial-theme.css`):
   - 粗细不均的机加工线条（粗/中/细/烧蚀/断裂多种类型）
   - 烧蚀线条：模糊 + stroke-dasharray 断裂效果
   - 热影响区线条：暗橙色 + blur 滤镜
   - 断裂线：不连续的虚线效果

2. **金属尘埃粒子系统** (`knowledge-ui.js`):
   - 6-8 个极暗金属尘埃（替代原来的 10-15 个亮粒子）
   - 颜色：暗青灰为主（65%），铁灰（27%），暗橙（8%）
   - 运动：缓慢向下沉降（15-25秒），像冷却金属粉尘
   - 尾迹：极短（1px），几乎不可见

3. **热应力波脉冲** (`knowledge-ui.js`):
   - 每 10-15 秒从随机网格节点扩散
   - 三色脉冲：青色（65%）、橙色（20%）、暗红（15%）
   - 效果：圆形边框向外扩散，模拟热应力波
   - 意图：暗示"档案库正在被实时维护/锻造"

4. **性能优化**:
   - 面板不可见时自动停止动画
   - 使用 MutationObserver 监听面板状态
   - 粒子自动清理，避免内存泄漏

**解决效果**:
- 网格从"数字感"变为"真实机加工金属表面"
- 粒子从"梦幻星尘"变为"高温余热工业废气"
- 脉冲效果增加活感和紧张感，暗示系统正在运行

## 2026-03-10

### 优化：知识库 UI 冷锻工业全息风格重构

**发现者**: Qwen Code

**问题描述**:
- 原知识库 UI 为"暗黑科技"风格，纯黑背景 + 简单六边形网格
- 缺少与 MechForge "锻造"主题的视觉关联
- 配色单一，缺少层次感和工业质感

**解决方法**:
1. **创建工业风主题样式** (`css/industrial-theme.css`):
   - 深海军蓝(#05070F)到近黑的极暗渐变底色
   - 碳纤维纹理 + 锻造划痕效果（使用 noise + linear gradient）
   - 粗细不均的机加工网格线条，模拟高温烧蚀金属表面
   - 钛合金灰(#1A252F)作为卡片底色

2. **更新配色方案**:
   - 主色：冷青(#00D4FF) - 保持科技感
   - 警示色：暗红(#ff4757) + 橙色(#ff6b35) - 用于 CRITICAL/WARNING 标签
   - 辅助色：钛合金灰系 - 营造工业金属质感

3. **多层辉光效果**:
   - 内层：锐利亮线（0 0 2px）
   - 中层：扩散雾（0 0 8px）
   - 外层：极淡晕（0 0 30px）
   - 警示辉光：红色/橙色多层发光

4. **更新知识库面板** (`index.html`):
   - 添加工业风背景层和 SVG 网格
   - 新增标签过滤器（ALL/MANUAL/SPEC/CASE/CRITICAL/WARNING）
   - 更新标题为 "KNOWLEDGE FORGE"
   - 工业风格搜索按钮和输入框

5. **优化交互逻辑** (`app/knowledge/knowledge-ui.js`):
   - 添加标签过滤功能
   - 工业风格加载动画
   - 模拟数据展示不同级别的结果项
   - 悬停效果和过渡动画

**解决效果**:
- 背景从"虚空"变为有"重工业历史感"的锻造车间底板
- 强化了 MechForge 的"锻造"主题
- 多层辉光效果提升了视觉层次感和科技感
- 警示色系的引入使重要信息更加醒目

## 2026-03-09

### 优化：模块化架构重构

**发现者**: Qwen Code

**问题描述**:
- `server.py` 文件过长（843行），包含所有 API 端点，维护成本高
- `ai-service.js` 中 `sendMessage` 方法未传递 `message` 参数导致 API 调用失败
- 缺少统一的错误处理机制
- 前端模块初始化缺少错误边界处理
- 缺少 `pyproject.toml` 配置文件

**解决方法**:
1. 将 `server.py` 拆分为模块化 API 结构：
   - `api/__init__.py` - 模块导出
   - `api/state.py` - 全局状态管理
   - `api/deps.py` - 依赖注入（懒加载）
   - `api/health.py` - 健康检查 API
   - `api/chat.py` - 聊天 API
   - `api/rag.py` - RAG API
   - `api/config.py` - 配置 API
   - `api/gguf.py` - GGUF 模型 API
   - `api/errors.py` - 统一错误处理

2. 修复 `ai-service.js` 中的 bug

3. 添加统一错误处理中间件

4. 优化前端模块初始化

5. 添加 `pyproject.toml` 配置

**解决效果**:
- 代码结构清晰，每个模块职责单一
- API 调用正常工作
- 错误响应格式统一
- 前端初始化更加健壮
- 项目配置标准化
