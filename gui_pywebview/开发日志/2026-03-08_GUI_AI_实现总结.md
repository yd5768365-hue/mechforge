## 2026年3月8日

### 新增功能：GUI 后端服务器与 AI 聊天集成
- **实现者**：Qwen Code
- **功能描述**：为 GUI 版本实现完整的后端服务器和 AI 聊天功能，借鉴 CLI 版本的逻辑
- **核心实现**：
  1. **后端服务器** (`gui/server.py`)：
     - FastAPI Web 服务器，提供 HTTP API
     - AI 聊天端点：`/api/chat`（非流式）、`/api/chat/stream`（流式）
     - RAG 检索端点：`/api/rag/search`、`/api/rag/status`
     - 配置管理端点：`/api/config`、`/api/models`
     - 健康检查：`/health`、`/status`
     - CORS 配置支持本地文件访问
     - 流式响应使用 Server-Sent Events (SSE)

  2. **前端服务层**：
     - `AIService` (`gui/services/ai-service.js`)：
       - 流式/非流式消息发送
       - RAG 检索
       - 对话历史管理
       - 配置加载
     - `APIClient` (`gui/core/api-client.js`)：
       - HTTP 请求封装
       - 本地文件模式支持
       - 错误处理

  3. **应用逻辑** (`gui/app.js`)：
     - 消息发送与显示
     - 流式响应实时更新
     - 事件总线集成
     - 界面交互处理

  4. **启动脚本**：
     - Windows：`gui/start.bat`
     - Linux/Mac：`gui/start.sh`
     - 自动检查依赖并安装

  5. **配置文件**：
     - `gui/server_config.yaml`：服务器配置
     - `gui/requirements.txt`：Python 依赖

  6. **文档**：
     - `gui/README.md`：完整文档
     - `gui/QUICKSTART.md`：快速启动指南
     - `gui/IMPLEMENTATION.md`：实现总结
     - `gui/test_server.py`：测试脚本

- **技术架构**：
  ```
  前端 (HTML/JS/CSS) ←HTTP→ 后端 (FastAPI) ←→ MechForge Core
  ```

- **核心特性**：
  1. **流式响应**：使用 SSE，实时显示 AI 回复，低延迟
  2. **RAG 集成**：多路召回（向量 + BM25），智能触发
  3. **对话管理**：自动历史记录，支持清空
  4. **配置灵活**：环境变量覆盖，动态配置加载

- **使用方法**：
  ```bash

  cd gui
  start.bat

  # Linux/Mac
  cd gui
  ./start.sh
  ```

- **API 端点**：
  - `GET /health` - 健康检查
  - `GET /status` - 系统状态
  - `POST /api/chat` - 非流式聊天
  - `POST /api/chat/stream` - 流式聊天
  - `GET /api/chat/history` - 获取历史
  - `DELETE /api/chat/history` - 清空历史
  - `POST /api/rag/search` - 搜索知识库
  - `GET /api/rag/status` - RAG 状态
  - `POST /api/rag/toggle` - 切换 RAG
  - `GET /api/config` - 获取配置
  - `POST /api/config` - 更新配置
  - `GET /api/models` - 可用模型列表

- **解决效果**：
  - GUI 版本具备完整的 AI 聊天功能
  - 流式响应提供流畅的用户体验
  - RAG 检索与 CLI 版本保持一致
  - 启动脚本简化部署流程
  - 完善的文档便于用户快速上手
